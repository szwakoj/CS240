    Unit 3 Test Review Questions

    Due: 5/7/26

    Description

    This HW will serve as a review of the material thus far and have questions to prepare you for the upcoming exam. This test is cumulative and will test materials from all of the posted lessons and on the in class code and lectures.

    The test will have True/False, Multiple Choice, Short Answer, Fill-in-the-Blank, and Complexity Analysis type questions. They will all be based on these short answer questions. Answer each to the best of your ability and use the material and in-class notes to fill in any gaps you find you have. This is meant to be a graded review - simply looking up answers without understanding will not prepare you for the exam. Try your best and ask questions during class time.

    Section 1: C++ Fundamentals - /30 pts

    1. (4 pts) | ~3-4 sentences C++ is often described as a "superset" of C. What does this mean in practical terms? What is the main compile command difference between C and C++, and what does this imply about existing C code?

    2. (4 pts) | ~3-4 sentences What is a namespace in C++? Why are they useful in large projects, and what is the purpose of the std:: prefix you see on standard library features like cout and cin?

    3. (4 pts) | ~2-3 sentences + example What does the using keyword do in C++? Write a short example showing the difference between using std::cout explicitly versus bringing it in with a using declaration. What is the risk of writing using namespace std; in a large project?

    4. (4 pts) | ~3-4 sentences Compare C++ I/O (std::cout / std::cin) to C I/O (printf / scanf). What is the key advantage of the C++ approach, and what are std::endl and the << operator doing in an output statement?

    5. (4 pts) | ~3-4 sentences What problem does std::string solve compared to C-style char arrays? List at least three operations std::string supports natively that required manual work or library functions in C. When would you still need to use a raw char array?

    6. (4 pts) | ~3-4 sentences Explain C++ references. How does passing by reference differ from passing by pointer in C, both in the function definition and at the call site? What are two limitations of references compared to pointers that make raw pointers still necessary?

    7. (4 pts) | ~3-4 sentences Compare new/delete in C++ to malloc/free in C. What is the key behavioral difference when allocating objects (not just primitives)? Why must you use delete[] instead of delete when freeing an array?

    8. (6 pts - 2 pts each) Match each term to its correct definition by filling in the table:

    Term Definition
    Constructor
    Destructor
    Access Control

    Definitions (use each once):

    • A mechanism in C++ classes that restricts which parts of a program can read or modify a member, using keywords like public and private
    • A special class function with no return type that shares the class's name and runs automatically when an object is created, used to initialize member data
    • A special class function prefixed with ~ that runs automatically when an object is destroyed or goes out of scope, used to release any resources the object owns

    Section 2: Non-Linear Data Structures - /20 pts

    1. (4 pts) | ~3-4 sentences What makes a data structure non-linear? How does this fundamentally differ from the linear structures covered in this course so far, such as arrays and linked lists? Give two real-world examples of data that is naturally non-linear and explain why forcing it into a linear structure would be a poor fit.

    2. (4 pts) | ~3-4 sentences What is a tree? Describe its hierarchical structure using the terms: root, parent, child, and leaf. Why is a tree considered non-linear even though you can traverse all of its nodes in a sequential order?

    3. (4 pts) | ~3-4 sentences What is a graph? How does it generalize the concept of a tree, and what two components formally define a graph? Give one example of a real-world system that is best modeled as a graph and explain what the nodes and edges represent in that context.

    4. (4 pts) | ~2-3 sentences each For each structure below, describe in your own words what kind of relationships it is best suited to represent and why a linear structure would fail to capture that relationship naturally:

    Structure What relationships does it model? Why linear structures fall short
    Tree
    Graph

    5. (4 pts) | ~3-4 sentences Non-linear structures introduce the concept of traversal strategies that do not exist in linear structures. Why does the branching nature of trees and graphs make the order in which you visit nodes a meaningful design decision? Name two broad categories of traversal strategy and describe in one sentence what distinguishes them from each other.

    Section 3: Trees and Binary Search Trees - /25 pts

    1. (4 pts) | ~3-4 sentences What is a binary tree? What constraint does "binary" place on the structure, and how does this differ from a general tree? Describe what information a single node in a binary tree must store compared to a node in a singly linked list.

    2. (4 pts) | ~3-4 sentences What additional property does a binary search tree (BST) enforce beyond just being a binary tree? State the ordering rule precisely in terms of a node and its left and right subtrees. Why does this property make searching significantly more efficient than scanning a linked list?

    3. (4 pts) | ~3-4 sentences A BST and a singly linked list are both node-based, pointer-linked structures. Under what condition does a BST effectively become a linked list? Describe the shape of the tree when this happens, explain what kind of insertion order causes it, and what it means for the performance of search.

    4. (4 pts) | ~2-3 sentences + example What is the height of a tree and why does it matter for BST performance? What is the height of a perfectly balanced BST with 7 nodes, and what is the height of the worst-case degenerate BST with the same 7 nodes? Write out what both trees would look like structurally.

    5. (4 pts) Fill in the time complexity table for BST operations under both average and worst case conditions and answer the follow-up question:

    Operation Average Case Worst Case
    Search
    Insert
    Delete
    Find minimum
    Find maximum

    a. (1 pt) What single word describes the worst-case shape of a BST, and what common data structure does it perform identically to in that state?

    6. (4 pts) | ~3-4 sentences What does it mean for a BST to be balanced? Why is balance the central problem that more advanced tree structures attempt to solve? Without going into implementation detail, describe in plain terms what an auto-balancing tree does differently than a basic BST when a new node is inserted.

    Section 7: C++ Implementation - /25 pts

    For all implementations, you MUST use C++. You do not need to handle edge cases beyond what is described. Focus on correctness of structure and logic over style.

    Add in these as DoubleLinkedList.hpp and BinarySearchTree.hpp alongside the answers in a zip file like normal.

    1. (12 pts) | Doubly Linked List

    Implement a doubly linked list that supports the following operations. Your implementation must include a Node struct/class and a LinkedList struct/class or equivalent wrapper.

    Required operations:

    • prepend(value) - insert a new node at the front
    • append(value) - insert a new node at the back
    • remove(value) - remove the first node with the given value
    • print() - print all values front to back

    2. (13 pts) | Binary Search Tree

    Implement a binary search tree that supports the following operations. Your implementation must include a Node struct/class and a BST struct/class or equivalent wrapper.

    Required operations:

    • insert(value) - insert a new value maintaining BST ordering
    • insert(vector) - sort an incoming vector using STL functions and insert it in a way that results in a balanced tree
    • search(value) - return true/1 if the value exists in the tree, false/0 otherwise
    • print_inorder() - print all values in sorted ascending order
